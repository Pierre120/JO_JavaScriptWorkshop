JavaScript Basic Syntax, Variable Declaration, Hoisting and Scopes

Basic Syntax

Code Blocks
- Indicated by curly braces { }.
Statements
- May be terminated with semicolons ; but this is highly encouraged.
Comments
- Code after double slashes // or between /* and */ is treated as a comment.
Declaring variables
1.) Variables must be declared before use.
let declaredVar = 101;
2.) Must use either var, let, or const to declare a variable.
var someVar = 'Hello!';
3.) Variable names are case sensitive.
declaredVar != declaredvar

Variable Declaration

Undeclared variables
- Using or referencing to an undeclared variable will throw a ReferenceError.
Sample Code:
undeclaredVariable = 10;
consol.log(undeclaredVariable); // will throw ReferenceError
Redeclaration of Variables
- A variable declared using the `var` keyword can be redeclared in the same scope.
Sample Code:
var redeclareVar = 'old';
var redeclareVar = 'new'; // valid
- A variable declared using the `let` and `const` keyword can't be redeclared in the same scope.
Sample Code:
let redeclareLet = 'old';
let redeclareLet = 'new'; // invalid
- Variables declared with any of the keywords can be redeclared in an inner, or nested scope or block.
Sample Code:
let globalLet = 'global';
function redeclareLet() { let globalLet = 'nested'; console.log(globalLet) // output: 'nested' }
console.log(globalLet); // output: 'global'

Hoisting

Hoisting is JavaScript's default behavior of moving all declarations to the top of the current scope (to the top of the current script or the current function).
**Only works with `var`
Sample Code:

varHoisting = 'Valid';
console.log(`Hoisted 'var' variable: ${varHoisting}`);
var varHoisting;
